# 더망고 상품정책 일괄 관리 매크로 사용법 (v2: 적용/변경/해제)

`admin_group.php`의 **모든 페이지·모든 검색필터**에 대해 상품정책을 일괄 **적용/변경/해제**합니다.
이미 정책이 적용된 필터의 **현재 정책**을 읽어, **"현재 정책 조건 + 필터이름 포함"** 으로 대상을 골라 처리할 수 있습니다.
(예: 이름은 `ABC마트`인데 `독일자라` 정책이 잘못 들어간 필터만 골라 `abc마트` 정책으로 변경하거나 해제)

## 동작 원리
- 각 필터의 현재 정책은 행의 드롭다운 `#select_{필터ID}`의 선택값으로 렌더링됩니다. 매크로는 모든 페이지를 fetch로 읽어 `select[id^=select_]` 기준으로 **필터ID·필터이름·현재 정책값**을 수집합니다.
  (이미 적용된 필터는 "상품정책적용" 앵커가 사라지고 드롭다운이 바로 보이므로, select 기준으로 열거해야 전체가 잡힙니다.)
- **적용/변경**: 사이트의 `config_mapping_save`와 동일하게 `admin_category_ok.php`에 `amode=group_config_mapping&uid={필터ID}&config_uid={정책값}` POST → 응답 "정책이 적용되었습니다."
- **해제(삭제)**: `config_mapping_remove`와 동일하게 `amode=group_config_mapping_remove&uid={필터ID}` POST → 응답 "정책이 해제되었습니다."
- 상품정책 옵션(기본설정 / 솔데글KR-abc마트 / 솔데글GL-독일자라 등)은 페이지에서 **동적으로 읽어** 드롭다운에 채웁니다(하드코딩 없음).

## 설치
1. Chrome에 Tampermonkey 확장 설치.
2. Tampermonkey → 대상 스크립트 내용 전체를 `더망고_상품정책_일괄적용.user.js`로 교체 후 저장(Ctrl+S). (기존 v1을 쓰고 있었다면 그 스크립트를 이 내용으로 덮어쓰면 됩니다.)
3. 더망고 관리자에 **로그인**한 상태로 `https://tmg4682.mycafe24.com/mall/admin/admin_group.php` 를 연다.

## 사용
페이지 우측 상단 **"상품정책 일괄 관리"** 패널:

1. **작업**: `적용/변경` 또는 `해제(삭제)` 선택.
2. **대상 조건**
   - **현재 정책**: `전체` / `미적용` / 특정 정책 중 선택. 선택한 현재 정책을 가진 필터만 대상이 됩니다.
   - **이름 포함**: 필터이름에 이 문구가 든 필터만 대상(공백 무시, 대소문자 무관). 비워두면 이름 조건 없음.
3. **변경할 정책**(작업이 `적용/변경`일 때만 표시): 새로 적용할 정책 선택. 이미 그 정책인 필터는 자동으로 제외됩니다.
4. 옵션: **테스트(저장 안 함)** 으로 대상만 세어보기, **앞에서 N개만**(0=전체) 으로 소량 확인.
5. **시작** → 확인창에서 조건·작업·대상 개수를 확인 후 진행. 진행 중 **정지** 가능.
6. 필터별 성공/실패 상세는 **콘솔(F12)** 에 기록됩니다. **처리 후 화면 반영은 새로고침** 해야 보입니다.

## 예시: ABC마트에 잘못 들어간 독일자라 정책 바로잡기
- 작업 = `적용/변경`
- 현재 정책 = `솔데글GL-독일자라`
- 이름 포함 = `ABC마트`
- 변경할 정책 = `솔데글KR-abc마트`
- (처음엔 **테스트** 또는 **N=3** 으로 대상 개수/동작 확인 후 전체 실행)

또는 정책을 아예 없애려면 작업을 `해제(삭제)` 로 두고 같은 대상 조건으로 실행합니다.

## 참고
- 목록 페이지네이션: `admin_group.php?...&ft_num=100&pg=N` (매크로가 끝 페이지까지 자동 순회). 검증 시 전체 202필터가 3페이지에 걸쳐 열거됨.
- 성공 판정: HTTP 200 + 응답 문구("적용되었습니다"/"해제되었습니다"). 실패 필터는 콘솔 로그의 `err`로 확인 후 재실행하면 됩니다.
- 변경/해제는 되돌릴 수 있습니다(다른 정책으로 다시 적용, 또는 해제 후 재적용).
