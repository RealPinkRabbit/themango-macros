# 카테고리매핑 규칙 가이드 (자동 파일기반 v1 · ② 매핑 단계 · 필터형/의류)

> 이 문서는 **Claude가 `카탈로그+매핑` 엑셀을 받아 `매핑` 시트를 채울 때 지키는 규칙**입니다.
> 규칙이 매크로 코드(정규식)에서 빠져 이 문서로 이동했습니다 → **신상품/새 필터가 추가돼도 매크로 수정 없이 이 규칙대로 Claude가 판단**합니다.
> 대상은 **검색필터형(독일자라 등 해외 브랜드) 의류·잡화 필터**입니다. (단계형=ABC마트 신발과 달리, 여기는 **의류 전 유형 + 11번가 해외 우선**.)

## 입력/출력
- 입력: 엑셀 워크북 2개 시트
  - `카탈로그`: `마켓 | 카테고리경로 | 코드` — 각 오픈마켓에서 **선택 가능한 의류·잡화 카테고리 전체**(현재 트리, 코드 포함). 이미 아동/반려동물/미용/비-패션은 걸러져 있음.
  - `매핑`: `필터ID | 필터명 | 성별 | 유형 | 상태 | 옥션 | G마켓 | 11번가 | 스마트스토어 | 쿠팡 | 롯데ON | LF몰 | 머스트잇 | 쇼피 | 큐텐JP | 플레이오토` — 각 필터(행)의 마켓 열은 비어 있음.
- 출력: `매핑` 시트의 각 마켓 칸에 **그 마켓 `카탈로그`에 존재하는 카테고리경로 문자열을 그대로** 기입. (코드는 매크로가 카탈로그로 자동 해석하므로 경로만 정확히.)
  - **반드시 카탈로그에 있는 경로를 그대로** 쓸 것(오타/변형 금지 → 매크로가 코드를 못 찾음).
  - 빈 칸으로 두면 = **그 마켓은 기존값 유지**(덮어쓰지 않음). 적절한 카테고리가 없으면 **빈 칸**으로 둠(억지 매핑 금지).
  - `상태` 열이 `완료`인 행은 이미 매핑돼 있음 — 다시 채우면 덮어씀. 미매핑만 처리하려면 그 행은 비워 둠(또는 수집 시 "미매핑만" 옵션 사용).

## 매핑 규칙 (우선순위)
각 필터 × 각 마켓에 대해, 그 마켓 카탈로그 후보 중 아래를 만족하는 **가장 근접한 카테고리**를 고른다. 필터의 상품유형은 `유형` 열(및 `필터명`)로 판단한다.

1. **성별 일치(필수)**: 필터 성별(남성/여성)과 반대 성별 카테고리 배제. `공용/유니섹스`는 허용. (성별이 '공용'이면 남/녀 무관.)
2. **11번가 = 해외 우선(필수)**: 대상은 **해외 브랜드**(독일자라 등) → 11번가는 `해외` 카테고리(`해외패션 > …`, `해외명품 > …` 등)를 우선. 국내 카테고리밖에 없으면 그때만 국내로. (※ 단계형 ABC마트는 반대로 국내 우선 — 혼동 주의.) 또한 11번가는 `디자이너`/`biz` 카테고리는 피한다.
3. **옥션·G마켓 = 브랜드 회피**: 경로가 `브랜드…`로 시작/포함되면 피하고 비-브랜드 일반 카테고리로.
4. **금지 영역 배제**: 어린이/유아/아동/키즈/주니어, 반려동물, 도서/e쿠폰/모바일/렌탈/배달음식, 출산/육아/임산부, 위생용품/의료기기/의약품은 배제(카탈로그에서 이미 걸러졌지만 재확인).
5. **유형(소분류) 최대한 일치** — 필터 유형 → 마켓 카테고리 대응(예시):
   - 상의: 티셔츠 → `티셔츠`, 셔츠/블라우스 → `셔츠/남방`·`블라우스/셔츠`, 니트 → `니트/스웨터`, 맨투맨/후드 → `맨투맨/스웨트셔츠`·`후드`, 폴로 → `카라티셔츠/피케` 또는 `티셔츠`.
   - 하의: 청바지 → `청바지/데님`, 반바지 → `반바지`·`숏팬츠`, 슬랙스/팬츠 → `슬랙스`·`면바지`, 레깅스 → `레깅스`.
   - 원피스/스커트: 원피스 → `원피스`, 점프수트 → `점프수트`·`원피스`, 스커트 → `스커트/치마`.
   - 아우터: 자켓/블레이저 → `재킷`·`블레이저`, 코트 → `코트`, 점퍼/패딩 → `점퍼`·`패딩/야상`.
   - 가방: `가방` 계열(백팩/토트/숄더/크로스/클러치)을 성별·형태에 맞춰.
   - 신발: 스니커즈 → `스니커즈/운동화`, 부츠 → `부츠/워커`, 샌들/슬리퍼 → `샌들/슬리퍼`, 구두/로퍼 → `구두`·`로퍼`, 힐 → `힐/펌프스`.
   - 잡화: 모자 → `모자`, 벨트 → `벨트`, 양말 → `양말`, 선글라스 → `선글라스/아이웨어`, 스카프 → `스카프/머플러`, 주얼리 → `주얼리/액세서리`, 넥타이 → `넥타이`.
   - 수영복 → `수영복/비치웨어`·`비키니`.
6. **"비슷하면 OK"**: 정확한 소분류 칸이 없으면 상위·근접 카테고리로. (예: '오버셔츠'는 셔츠 칸이 없으면 `남방/셔츠`, '카디건'은 니트 칸으로.)
7. **비-의류 오분류 금지**: 필터 유형과 다른 대분류(예: 티셔츠 필터에 가방/신발 카테고리)를 고르지 않는다.
8. **적합 후보가 전혀 없으면 빈 칸**(미매핑). 반대성별/브랜드/금지 카테고리밖에 없다면 채우지 말 것.

## 체크(자기검증)
매핑을 마치면 스스로 확인: ①모든 값이 그 마켓 카탈로그에 **실제로 존재**하는가, ②성별 위반 없는가, ③11번가가 **해외** 우선인가(해외 후보가 있는데 국내를 고르지 않았는가)·`디자이너/biz` 아닌가, ④옥션/G마켓에 `브랜드` 없는가, ⑤필터 유형과 다른 대분류로 오분류 없는가.

## 진행 방식
- 카탈로그 후보가 많아도 **마켓별로 후보를 필터→최근접 1개**를 고른다. 모든 필터 × 11마켓을 같은 규칙으로 일관 처리.
- 결과 엑셀(`매핑` 시트 채운 것)을 저장해 매크로 ③ 적용에 넣으면 그대로 저장된다.
- **고시(상품고시정보) 처리는 매크로가 자동**으로 한다(의류=`의류`, 잡화=`패션잡화`, 신발=`구두/신발` 상품군 + 11번가·롯데ON "상품상세페이지 참조"). Claude는 카테고리 경로만 채우면 된다.
