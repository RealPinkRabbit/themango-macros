# 더망고 카테고리매핑 자동 · 파일기반 v1 사용법 (수집 → Claude매핑 → 적용)

## 왜 바꿨나
기존 `더망고_카테고리매핑_자동.user.js`는 **매크로 안의 정규식 규칙**으로 카테고리를 자동 선택했습니다. 그래서 신상품/새 카테고리가 나올 때마다 규칙을 다시 손봐야 했습니다. 이 버전은 **규칙을 매크로에서 빼내** 단계형 파일기반과 동일한 3단계로 분리합니다.

1. **수집(Export)** — 매크로가 마켓별 "선택 가능한 의류·잡화 카테고리 전체(카탈로그: 경로+코드)"와 대상 필터 목록을 **엑셀**로 내려받음.
2. **매핑** — 그 엑셀을 **Claude**에게 주면, `매핑규칙.md`대로 각 필터×마켓에 카탈로그 경로를 채워 돌려줌.
3. **적용(Apply)** — 그 엑셀을 매크로에 넣으면, 카탈로그로 경로→코드를 해석해 필터마다 설정페이지(`ps_ftid`)에서 **주입·고시·저장**. 저장 후 **되읽기 검증**.

→ 신상품이 추가돼도 **매크로 수정 없이** Claude가 같은 규칙으로 판단합니다. 매핑 근거가 엑셀로 남아 검수·수정이 쉽습니다.

## 대상 (기존 자동 매크로와 동일)
- **검색필터형** 필터(독일자라 등 해외 브랜드). ABC마트는 **단계형**(`더망고_카테고리매핑_단계형_파일기반.user.js`)을 쓰세요.
- 필터형은 11번가 **해외 우선**입니다(단계형 ABC=국내 우선과 반대 — 매핑규칙 참조).

## 설치
1. Tampermonkey에서 새 스크립트 → `더망고_카테고리매핑_자동_파일기반.user.js` 붙여넣기 → 저장.
   - 상단 `@require`로 **SheetJS(xlsx)** 를 불러옵니다(엑셀 읽기/쓰기). 첫 실행 시 Tampermonkey가 외부 스크립트 허용을 물으면 허용하세요.
2. 더망고 관리자에 로그인 상태.
3. `정책적용 > 상품수집조건(검색필터) 관리 및 정책적용`(`admin_group.php`)으로 이동하면 우측 상단 보라색 패널이 뜹니다.

## ① 수집(Export)
1. 패널에서 **대상 라벨** 확인(기본 `독일자라`). 필터명에 이 문자열이 포함된 필터만 대상이 됩니다.
   - **미매핑만** 체크: `설정하기`(아직 매핑 안 함) 필터만. 해제 시 `설정수정`(완료)까지 전체.
2. **[카탈로그+대상 엑셀 내보내기]** 클릭.
   - 대상 필터를 모은 뒤, 마켓 검색이 가능한 **설정페이지로 자동 이동**해 11개 마켓의 의류·잡화 카테고리를 키워드 스윕으로 모읍니다(마켓당 수십 초, 전체 **수 분**).
   - 끝나면 `카테고리매핑_독일자라_날짜.xlsx`가 다운로드됩니다. 시트 2개: `카탈로그`(마켓·경로·코드), `매핑`(필터 행 + 성별/유형/상태 + 빈 마켓 열).
3. 진행 중 뜨는 팝업/페이지 이동은 건드리지 마세요.

## ② 매핑 (Claude)
- 다운로드된 엑셀을 Claude(코워크)에게 주고 "매핑규칙대로 매핑 시트를 채워줘"라고 하면 됩니다(`더망고_카테고리매핑_자동_파일기반_매핑규칙.md` 기준).
- Claude는 각 필터×마켓 칸에 **그 마켓 카탈로그에 있는 경로**를 채웁니다(성별 일치·11번가 해외·브랜드/비-의류 배제·최근접 유형).
- 결과 엑셀을 저장합니다. (사람이 직접 검수·수정도 가능 — 칸의 경로를 카탈로그의 다른 경로로 바꾸면 됨. 빈 칸=그 마켓 유지.)

## ③ 적용(Apply)
1. 목록 페이지 패널에서 **[파일 선택]** → 매핑 완료된 `.xlsx` 선택.
2. **[매핑파일대로 적용 시작]** 클릭 → 확인창.
   - 매핑이 **하나라도 채워진 필터만** 순차 처리합니다(빈 필터는 건너뜀). 각 필터 설정페이지(`?tm=F&ps_ftid=…`)로 이동 → 지정 코드 주입 → 고시 상품군/참조 처리 → 저장.
   - **빈 칸인 마켓은 기존 저장값을 그대로 유지**합니다(덮어쓰지 않음).
   - 고시는 매크로가 자동 처리(상품군 지정 + 11번가·롯데ON "상품상세페이지 참조"). 특정 마켓이 저장검증에 계속 걸리면 **그 마켓만 비우고(no_category)** 나머지를 저장해 필터 통째 스킵을 막습니다(로그에 `dropped` 기록).
3. **처음엔 소수만 테스트** 권장: 매핑 엑셀에서 필터 2~3개만 마켓 칸을 채우고(나머지는 비워둠) 적용 → 아래 검증으로 확인 후 전체 진행.
4. **[정지]** 로 현재 항목까지 마치고 멈춤. 진행은 localStorage로 새로고침을 넘어 이어집니다.

## 검증 (되읽기)
- 목록 페이지 패널 **[실제 저장값 되읽기 검증]** → 대상 필터마다 실제 저장값을 서버에서 되읽어 CSV + 이상리포트(미매핑/사이트무효) 다운로드.
- 필터형은 `ps_ftid`로 서버 스코핑돼 **되읽기가 신뢰 가능**합니다(단계형 세트페이지는 되읽기 불가라 별도 방식). **[A] "카테고리를 변경해주세요"(무효)** 는 저장 후에만 서버가 판정하므로 적용 후 반드시 확인하세요.

## 주의
- 카탈로그는 **현재 마켓 트리에서 검색된 것**이라 코드가 항상 유효 → "변경해주세요" 무효 저장을 원천 차단합니다.
- 새 필터/새 카테고리가 생기면 **①수집만 다시** 하면 카탈로그·대상이 갱신됩니다(규칙/코드 수정 불필요).
- 카탈로그에 원하는 카테고리가 안 잡히면(마켓 트리에 없거나 스윕 키워드 밖) 해당 칸은 빈 칸으로 남습니다 → 필요 시 스크립트의 스윕 키워드(`SWEEP_KR`)에 한 단어 추가.
- 수집이 너무 오래 걸리면 `SWEEP_KR` 키워드 수를 줄이세요(대신 세부 카테고리가 일부 누락될 수 있음).
- 기존 규칙기반 매크로(`더망고_카테고리매핑_자동.user.js`)는 참고/폴백용으로 남겨둡니다. **다른 localStorage 키**(`tmg_catmap_auto_file_v1`)를 써서 공존 가능합니다.
